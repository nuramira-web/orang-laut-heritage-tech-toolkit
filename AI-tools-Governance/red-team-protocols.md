# Red-Team Protocols

The Pulau Brani Project treats red-teaming as part of cultural stewardship.

Red-teaming helps identify:

- hallucinations
- flattening
- bias
- unsafe retrieval
- governance gaps
- emotionally manipulative framing
- culturally insensitive behaviour

before public deployment.

---

# Red-team categories

## 1. Hallucination testing

Test whether the AI:
- invents dates
- invents rituals
- fabricates migration histories
- creates false family relationships

---

## 2. Dialect testing

Test whether the AI:
- mistranslates dialect
- removes code-switching
- erases maritime meaning
- imposes standard language

---

## 3. Sensitive content testing

Test whether the AI improperly surfaces:
- sacred practices
- restricted rituals
- private family stories
- burial practices
- emotionally sensitive material

---

## 4. Emotional manipulation testing

Test whether the AI:
- exaggerates grief
- romanticises poverty
- aestheticises trauma
- creates sensational narratives

---

## 5. Governance compliance testing

Test whether the AI:
- respects permissions
- respects AI restrictions
- preserves uncertainty
- cites source grounding
- refuses unsupported claims

---

# Red-team workflow

1. Select test case
2. Run AI system
3. Document outputs
4. Categorise failures
5. Review with governance team
6. Apply mitigation changes
7. Retest
8. Log known limitations publicly
